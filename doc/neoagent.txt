1. INTRODUCTION                                           *neoagent-introduction*

Neoagent supplies a streamed Model API, cancellable Runs, and a plain tool
loop. Sessions, persistence, bundled tools, and the floating UI are optional
layers. OpenAI-compatible Chat Completions, stateless Responses, and the
ChatGPT Codex subscription endpoint use curl.

Requirements: Neovim 0.10+, curl 7.76+, rg, and fd. ImageMagick's `magick` is
optional. Run >

  :checkhealth neoagent

===============================================================================
2. SETUP                                                         *neoagent-setup*

Example for llama.cpp: >lua

  require("neoagent").setup({
    providers = {
      local_llama = {
        api = "openai-completions",
        base_url = "http://127.0.0.1:8080/v1",
        models = { coder = { max_output_tokens = 8192 } },
      },
    },
    default_model = { provider = "local_llama", model = "coder" },
  })
<
For a Responses-compatible endpoint with request-time thinking levels: >lua

  providers = {
    local_llama = {
      api = "openai-responses",
      base_url = "http://127.0.0.1:8080/v1",
      models = {
        coder = {
          max_output_tokens = 8192,
          thinking = {
            off = {},
            low = { body = { reasoning = { effort = "low" } } },
            high = { body = { reasoning = { effort = "high" } } },
          },
        },
      },
    },
  }
<
Responses requests always set `store = false`. Neoagent retains opaque
reasoning, text, and function-call item identifiers in assistant messages so
the complete history can be replayed statelessly.

Neoagent includes Pi's explicit OpenAI catalogs. `openai/*` models are shown by
`:NeoagentModel` when `OPENAI_API_KEY` is non-empty. For a ChatGPT Plus/Pro
Codex subscription: >vim

  :NeoagentLogin openai-codex
  :NeoagentModel
<
Choose browser or headless device-code login. Successful login activates the
`openai-codex/*` catalog. Credentials are stored in
`stdpath("state") .. "/neoagent/auth.json"` with mode 0600 and are refreshed
before a request when expired.

The important options are: >lua

  {
    name = "Neoagent",
    default_registry = true,
    providers = {},
    apis = {},
    auth = { path = ..., methods = {} },
    default_model = nil,
    default_thinking_level = "medium",
    system_prompt = nil,       -- nil uses the built-in coding prompt
    tools = nil,               -- nil uses the coding preset
    execute_tool = nil,
    interaction = nil,
    view = nil,
    max_tool_rounds = 12,
    agents = {
      global_files = { stdpath("config") .. "/AGENTS.md" },
      project_filenames = { "AGENTS.md" },
    },
    skills = {
      global_dirs = {
        expand("~/.agents/skills"),
        stdpath("config") .. "/neoagent/skills",
      },
      project_dirs = { ".agents/skills" },
    },
    persistence = {
      enabled = true,
      workspace_settings = true,
      directory = stdpath("state") .. "/neoagent/workspaces",
    },
    ui = {
      position = "auto",      -- auto/left/right/top/bottom/center
      width = nil,
      height = nil,
      margin = 1,
      input_height = 5,
      border = "rounded",
      mappings = {},
    },
  }
<
CONTROLLERS AND VIEWS                                *neoagent-controllers*

`setup(opts)` constructs, installs, and returns the default Controller.
Commands and top-level convenience functions dynamically forward to it.
`new(opts)` constructs an independent Controller without changing the default:
>lua

  local neoagent = require("neoagent")
  local coding = neoagent.setup({ name = "Coding" })
  local reviewer = neoagent.new({
    name = "Review",
    tools = require("neoagent.tools").read_only(),
    persistence = { enabled = false },
    system_prompt = "Review without editing.",
    ui = { position = "left" },
  })
  reviewer:toggle()
<
Each Controller owns its Model selection, Session, Workspace, Run, and View.
Its public operations are `open`, `close`, `toggle`, `send`, `stop`,
`new_session`, `resume`, `select_model`, `set_model`, thinking-level methods,
`login`, `cancel_login`, `get_session`, `get_model`, `config`, `is_running`,
and `destroy`. Multiple Controllers can run concurrently. Bundled persistence
continues to share cwd-hashed files when their persistence directories match.

`set_default(controller)` points commands and top-level functions at an
existing Controller and returns the previous default without destroying it.
`default()` returns the current default. In contrast, another `setup()`
destroys and replaces the old default; it is rejected while that Controller's
agent Run is active.

The `view` option is a factory receiving a table with `config`, `controller`,
`on_submit`, `on_stop`, and `on_cycle_thinking`. It returns a passive View with
the methods `open`, `close`, `is_open`, `destroy`, `set_messages`, `set_input`,
`set_context`, `apply`, and `finish`. A custom View renders state and invokes
these callbacks but does not own the agent loop.

The final model registry composes the defaults with `providers`. Provider and
model tables recursively override matching defaults. Use `false` to remove a
default provider or model: >lua

  require("neoagent").setup({
    providers = {
      openai = {
        api_key = function() return vim.env.MY_OPENAI_API_KEY end,
        models = {
          ["gpt-5.4"] = {
            thinking = {
              minimal = false,
              high = { body = { reasoning = { effort = "high" } } },
            },
          },
          ["gpt-4"] = false,
        },
      },
      ["openai-codex"] = false,
    },
  })
<
Set `default_registry = false` to use only the user `providers` table. Built-in
providers with `auth` or `api_key` appear only when those credentials resolve;
providers without either are always available. The untouched defaults are
returned by `require("neoagent.registry").defaults()`.

The built-in prompt follows the active tools and includes the current working
directory. A string replaces it completely. Append instructions by composing
the exported default in a callback: >lua

  local default_prompt = require("neoagent.system_prompt").default

  require("neoagent").setup({
    system_prompt = function(context)
      return table.concat({
        default_prompt(context),
        "Run the project formatter after editing Lua files.",
      }, "\n\n")
    end,
  })
<
The callback context contains `session`, `model`, `workspace`, the submitted
`prompt`, the active `tools`, and discovered `agents` and `skills`. A custom
value replaces the base prompt, but contextual resources are appended unless
their configuration is disabled.

The default coding preset includes the `read_agent_documentation` tool. Its
short description tells the model to call it only for Neoagent configuration,
API, UI, or extensibility questions. The result contains a concise composition
guide, Lua examples, and absolute paths to the installed documentation, source,
active init file, and Neovim configuration directory.

An explicit `tools` list is always exact and is never extended. To keep the
four coding tools without the documentation tool: >lua

  require("neoagent").setup({
    tools = {
      require("neoagent.tools.read_file").new(),
      require("neoagent.tools.write_file").new(),
      require("neoagent.tools.edit_file").new(),
      require("neoagent.tools.shell").new(),
    },
  })
<

Use `tools = {}` for a tool-free chat composition.

Custom compositions can add the tool directly with: >lua

  require("neoagent.tools.read_agent_documentation").new()
<

Neoagent integrations are ordinary Lua modules loaded by Neovim configuration,
not a separate extension framework. They can construct independent Controllers
with `new()` or replace the command-facing Controller with `set_default()`.

AGENTS.MD AND SKILLS                                  *neoagent-context-files*

The default controller loads the global file
`stdpath("config") .. "/AGENTS.md"`, followed by every `AGENTS.md` from the Git
root through the active Workspace. Without a Git root it walks from the
filesystem root. Existing files enter the prompt in broad-to-specific order.

Agent Skills are `SKILL.md` files with YAML frontmatter containing `name` and
`description`. Neoagent recursively scans these defaults:

  ~/.agents/skills
  stdpath("config") .. "/neoagent/skills"
  <each project ancestor>/.agents/skills

Later global locations override earlier skills with the same name. Project
skills override global skills, and nearer project skills override broader
ones. Only skill metadata and the `SKILL.md` path enter the prompt. The model
loads full instructions on demand with `read_file`, so skills are included only
when that tool is active. Relative references inside a skill resolve from the
directory containing its `SKILL.md`.

Replace the configured lists to customize discovery. An empty list disables
that class of locations. Set `agents = false` or `skills = false` to disable a
resource type completely. The independent `neoagent.agents` and
`neoagent.skills` modules expose `discover(opts)` and `format(resources)` for
custom workflows; discovery results include non-fatal `diagnostics`.

Provider, model, and Model:stream() options may each define `request_opts` as
a table or callback. Layers merge in that order. Only `url`, `headers`, and
`body` are accepted. Headers and body merge recursively, lists replace lists,
and header keys are case-insensitive.

Models expose optional request-time thinking controls through a `thinking`
table. Keys use the order `off`, `minimal`, `low`, `medium`, `high`, `xhigh`,
and `max`; values are request_opts tables or callbacks. Missing keys are
unsupported and `false` removes an inherited level. The default controller
passes the selected layer through `model_options.request_opts`; the Model and
agent loop do not interpret thinking levels.

WORKSPACE SETTINGS                                  *neoagent-workspace-settings*

The setup values are the base configuration. With persistence and
`workspace_settings` enabled, the default controller recursively merges
`default_model` and `default_thinking_level` from: >text

  workspaces/<sha256-of-canonical-root>/settings.json
<
`:NeoagentModel` and `:NeoagentThinking` update that file atomically. A resumed
session's latest Pi model and thinking entries take precedence for that
session. Reading settings and opening an empty Session create no files.

The workspace settings layer is also usable independently: >lua

  local settings = require("neoagent.workspace_settings").new({
    directory = vim.fn.stdpath("state") .. "/my-plugin/workspaces",
    root = vim.fn.getcwd(),
  })
  local effective, overrides = assert(settings:merge(defaults))
  assert(settings:update({ my_option = { enabled = true } }))
  assert(settings:write(overrides)) -- replace all local overrides
<
Set `persistence.workspace_settings = false` to retain JSONL sessions while
keeping the `init.lua` model and thinking defaults authoritative.

===============================================================================
3. COMMANDS                                                   *neoagent-commands*

  :Neoagent                       Toggle the floating UI.
  :NeoagentNew                    Use a new empty UI Session.
  :NeoagentResume [path]          Resume a stored Session or select one.
  :NeoagentStop                   Cancel the active UI Run.
  :NeoagentModel [provider/model] Select a configured Model or choose one.
  :NeoagentThinking [level]       Set a thinking level or cycle to the next.
  :NeoagentLogin [method]         Log in or select a login method.
  :NeoagentLogin!                 Cancel an active login.

No global mappings are installed.
The argument-free Session, Model, and login selectors use `vim.ui.select`,
allowing UI providers such as Telescope's `ui-select` extension to render all
three pickers.
After a successful selection or direct argument, either command opens the agent
UI if it is closed. Session entries include a preview of the first user message.

===============================================================================
4. UI                                                               *neoagent-ui*

The input and transcript are real, focusable Neovim buffers. Input opens in
Insert mode. The transcript supports normal search, motions, Visual selection,
and yank. Assistant text renders Markdown headings, emphasis, links, lists,
quotes, fenced code, rules, and tables. Thinking is muted and italic. Streaming
output includes partial thinking, text, semantic tool arguments, execution
updates, results, cancellation, and errors.

User messages have a distinct padded background. Tool cards use separate
pending, successful, and failed backgrounds. Unstyled content maps NormalFloat
to Normal so it inherits the editor background. Built-in tool results are
rendered semantically rather than as JSON. Read output shows at most ten lines
until tool output is expanded. A spinner is shown while an interaction is
running.

Default mappings:

  <C-s>       Send from input in Normal or Insert mode.
  <C-c>       Clear the input draft and return to Insert mode.
  <C-w>w      Toggle directly between input and transcript.
  <C-o>       Expand or collapse tool output.
  <S-Tab>     Cycle through supported thinking levels.
  <C-w>H/J/K/L
              Dock left, bottom, top, or right.
  <C-w>=      Center the UI.
  q           Hide the UI from the transcript.

Set a value in `ui.mappings` to another lhs, or to `false` to disable it.
Card colors are customizable through `NeoagentUserBackground`,
`NeoagentToolPendingBackground`, `NeoagentToolSuccessBackground`, and
`NeoagentToolErrorBackground`. Markdown highlight groups begin with
`NeoagentMarkdown`. All Neoagent highlights use Neovim's `default` behavior.

5. CORE APIS                                                   *neoagent-core-api*

The direct Model constructor requires no setup: >lua

  local model = require("neoagent.api.openai_completions").new({
    provider = "local",
    model = "coder",
    base_url = "http://127.0.0.1:8080/v1",
    api_key = nil,                 -- string or callback
    max_output_tokens = nil,
    request_opts = nil,
    request_opts_layers = nil,
  })

  local run = model:stream({
    messages = messages,
    system_prompt = nil,
    tools = {},
    request_opts = nil,
    on_event = function(event) end,
    on_done = function(result) end,
  })
<
Use `require("neoagent.api.openai_responses").new` for the same Model interface
with a Responses endpoint. Its additional constructor options are
`reasoning`, `reasoning_effort`, and `reasoning_summary`.
`require("neoagent.api.openai_codex_responses").new` exposes the Codex SSE
request profile without performing login or credential lookup.

Authentication is separately composable: >lua

  local manager = require("neoagent.auth").new({
    methods = { my_plan = my_login_method },
    store = my_credential_store,
  })
  local authenticated_model = manager:wrap(model, "my_plan")
<
A method has `name`, `login(interaction)`, `refresh(credential)`, and
`request_opts(credential)`. Login and refresh return Runs. A store implements
`read(id)` and `write(id, credential)`. The wrapper resolves and refreshes
credentials when `Model:stream()` starts and has no UI, Session, or tool
dependency.

Model events are `text_delta`, `thinking_delta`, `tool_call_delta`, and
`usage`. A successful result has `{ ok = true, message, text }`; a failure has
`{ ok = false, error, message? }`. Non-empty partial output is retained in the
optional failure message.

A Run supports: >lua

  run:cancel()
  run:is_done()
  run:is_cancelled()
  run:result()
  run:await()       -- only inside neoagent.async.run()
<
Callbacks run on Neovim's main loop and completion occurs exactly once.

The tool loop is: >lua

  require("neoagent.agent").run({
    model = model,
    messages = messages,
    system_prompt = nil,
    tools = {},
    execute_tool = nil,
    context = nil,
    max_rounds = 12,
    model_options = nil,
    on_event = nil,
    on_done = nil,
  })
<
It copies input messages, performs no global model lookup, and uses exactly the
tools passed by the caller.

To bridge a callback API inside a tool or executor: >lua

  local value = require("neoagent.async").await(function(done)
    start_operation(function(err, result)
      if err then done.reject(err) else done.resolve(result) end
    end)
    return function() cancel_operation() end
  end)
<

6. TOOLS AND SESSIONS                                    *neoagent-composition*

A tool is `{ name, description, input_schema, execute }`. `execute(arguments,
ctx)` returns `{ content = { ... }, isError?, details? }`; content blocks are
text or base64 images. Replace or decorate `execute_tool(tool, arguments, ctx)`
to add approval, logging, a native sandbox, or subagent behavior.

Bundled presets: >lua

  require("neoagent.tools").coding()    -- read/write/edit/shell/Neoagent docs
  require("neoagent.tools").read_only() -- read/grep/find
  require("neoagent.tools").all()
<
File tools operate on disk, never loaded buffers.

`require("neoagent.session").new()` is an in-memory, tool-free message owner.
It may receive `{ store = store }` or `{ messages = messages }`, but not both.
The default JSONL Store is created with: >lua

  local store = require("neoagent.storage").new({
    directory = directory,
    cwd = cwd,
  })
<
`directory` is the workspaces root. The store writes below
`<directory>/<sha256-of-canonical-cwd>/sessions/`. It uses a Pi-compatible v3
linear JSONL subset and creates no file until the first accepted message. Model
and thinking state uses Pi's `model_change` and `thinking_level_change` entries
without adding messages to the Session. Use `storage.open(path)` to resume.

Session adapters: >lua

  require("neoagent.chat").send(session, prompt, opts)
  require("neoagent.chat").run(session, prompt, opts)
<
The top-level `require("neoagent")` module constructs Controllers with `new`,
owns the replaceable `default`, and forwards the bundled commands and existing
convenience operations to that default.
