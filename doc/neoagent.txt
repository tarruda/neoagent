1. INTRODUCTION                                           *neoagent-introduction*

Neoagent supplies a streamed Model API, cancellable Runs, and a plain tool
loop. Sessions, persistence, bundled tools, and the floating UI are optional
layers. Anthropic Messages, OpenAI-compatible Chat Completions, stateless
Responses, and the ChatGPT Codex subscription endpoint use curl.

Requirements: Neovim 0.10+, curl 7.76+, rg, and fd. ImageMagick's `magick` is
optional. Run >

  :checkhealth neoagent

===============================================================================
2. SETUP                                                         *neoagent-setup*

Example for llama.cpp: >lua

  require("neoagent").setup({
    providers = {
      local_llama = {
        api = "openai-completions",
        base_url = "http://127.0.0.1:8080/v1",
        models = {
          coder = { context_window = 131072, max_output_tokens = 8192 },
        },
      },
    },
    default_model = { provider = "local_llama", model = "coder" },
  })
<
Model requests use canonical JSON object ordering, including replayed tool-call
arguments. Process restarts preserve otherwise unchanged local prompt-cache
prefixes across persisted conversation resumes.

For a Responses-compatible endpoint with request-time thinking levels: >lua

  providers = {
    local_llama = {
      api = "openai-responses",
      base_url = "http://127.0.0.1:8080/v1",
      models = {
        coder = {
          max_output_tokens = 8192,
          thinking = {
            off = {},
            low = { body = { reasoning = { effort = "low" } } },
            high = { body = { reasoning = { effort = "high" } } },
          },
        },
      },
    },
  }
<
Responses requests always set `store = false`. Neoagent retains opaque
reasoning, text, and function-call item identifiers in assistant messages so
the complete history can be replayed statelessly.

Neoagent includes Pi's explicit OpenAI catalogs. Run `:NeoagentLogin openai`
to enter an API key through a masked prompt, or set `OPENAI_API_KEY` before
starting Neovim. Either source makes `openai/*` models available.

Anthropic uses the official Messages endpoint through two catalogs. Run
`:NeoagentLogin anthropic` or set `ANTHROPIC_API_KEY` for metered API-key
billing and `anthropic/*` models. Run `:NeoagentLogin anthropic-plan` for the
Claude Pro/Max browser or manual-code OAuth flow, or set an existing
`ANTHROPIC_OAUTH_TOKEN`, to enable the matching `anthropic-plan/*` catalog.
The catalogs contain Claude Opus 4.1, 4.5, 4.6, 4.7, and 4.8; Sonnet 4.5, 4.6,
and 5; Haiku 4.5; Fable 5; and their listed dated aliases.

Adaptive models expose their provider-supported effort levels through `low`,
`medium`, `high`, `xhigh`, and `max`. Budget-thinking models expose `off`,
`minimal`, `low`, `medium`, and `high` with budgets from 1024 to 16384 tokens.
Requests use ephemeral prompt-cache breakpoints, eager tool-input streaming,
signed thinking replay, and normalized cache usage. The plan composition adds
Claude Code identity and OAuth headers. Anthropic currently bills third-party
subscription OAuth requests as extra usage per token; included Claude plan
limits do not cover these requests. Select API-key billing by default with: >lua

  require("neoagent").setup({
    default_model = { provider = "anthropic", model = "claude-sonnet-4-6" },
  })
<
The built-in DeepSeek catalog uses the official OpenAI-compatible endpoint. Run
`:NeoagentLogin deepseek` or set `DEEPSEEK_API_KEY` to make
`deepseek/deepseek-v4-flash` and `deepseek/deepseek-v4-pro` available.
Both DeepSeek models declare a 1M-token context window, a 384k-token output
limit, and `off`, `high`, and `max` thinking levels. Streamed
`reasoning_content` is retained and replayed during tool turns. Select one by
default with: >lua

  require("neoagent").setup({
    default_model = { provider = "deepseek", model = "deepseek-v4-flash" },
  })
<
Z.AI provides a metered API and a global Coding Plan. Run `:NeoagentLogin zai`
or set `ZAI_API_KEY` to enable both catalogs. The `zai` provider uses
`https://api.z.ai/api/paas/v4` and contains `glm-4.5`, `glm-4.5-air`,
`glm-4.5-flash`, `glm-4.5v`, `glm-4.6`, `glm-4.6v`, `glm-4.7`,
`glm-4.7-flash`, `glm-4.7-flashx`, `glm-5`, `glm-5-turbo`, `glm-5.1`,
`glm-5.2`, and `glm-5v-turbo`. The `zai-coding-plan` provider uses
`https://api.z.ai/api/coding/paas/v4` and contains `glm-4.5-air`, `glm-4.7`,
`glm-5-turbo`, `glm-5.1`, `glm-5.2`, and `glm-5v-turbo`.

GLM-5.2 declares a 1M-token context window and exposes `off`, `low`, `medium`,
`high`, and `max` thinking profiles; its enabled profiles map to the provider's
`high` and `max` efforts. The other models expose toggle profiles from `off`
through `high`. GLM-4.5V, GLM-4.6V, and GLM-5V-Turbo accept image input. Z.AI
reasoning is retained across tool turns, supported models request streamed tool
calls, and streamed usage is normalized. Select a Coding Plan model by default
with: >lua

  require("neoagent").setup({
    default_model = { provider = "zai-coding-plan", model = "glm-5.2" },
  })
<
For a ChatGPT Plus/Pro Codex subscription: >vim

  :NeoagentLogin openai-codex
  :NeoagentModel
<
Choose browser or headless device-code login. Successful login activates the
`openai-codex/*` catalog. Credentials are stored in
`stdpath("state") .. "/neoagent/auth.json"` with mode 0600 and are refreshed
before a request when expired. Stored API keys use the same private credential
file and do not expire. A stored credential takes precedence over an ambient
API key. `:NeoagentLogout [method]` removes the stored value under the store
lock; environment variables remain unchanged and become active again.

The bundled GPT-5.6 Codex models use the
official Responses Lite request profile: tools and instructions are leading
developer input items, the Codex Lite header is set, reasoning context covers
all turns, reasoning summaries are omitted by default, and the default coding
prompt asks for commentary updates plus a final answer through Codex channels.

Codex retries transient transport, HTTP 429/5xx, and stream failures with
cancellable exponential backoff. Requests without output receive up to four
retries. The default Controller replays a retryable failed turn up to five
times and removes any failed assistant message from the active branch.

Configured Codex failures are appended as private JSONL diagnostics to
`stdpath("state") .. "/neoagent/codex.log"`. The 1 MiB log rotates to
`codex.log.1`. Records contain error classification, attempt counts, HTTP
status, curl exit code, request ID, and Cloudflare ray ID. Credentials, request
and response bodies, full header collections, and conversation content are
excluded. Configure or disable the log on the provider: >lua

  providers = {
    ["openai-codex"] = {
      diagnostics = { path = vim.fn.stdpath("state") .. "/my-codex.log" },
      -- diagnostics = false,
    },
  }
<

The important options are: >lua

  {
    name = nil,                -- Neo label and workspace-settings scope
    default_registry = true,
    providers = {},
    apis = {},
    auth = { path = ..., methods = {} },
    default_model = nil,
    default_thinking_level = "medium",
    compaction = {
      auto = true,
      reserve_tokens = 16384,
      keep_recent_tokens = 20000,
      run = nil,
    },
    system_prompt = nil,       -- nil uses the built-in coding prompt
    tools = nil,               -- nil uses the coding preset
    execute_tool = nil,
    interaction = nil,
    view = nil,                -- bundled Window's View constructor
    agents = {
      global_files = { stdpath("config") .. "/AGENTS.md" },
      project_filenames = { "AGENTS.md" },
    },
    skills = {
      global_dirs = {
        expand("~/.agents/skills"),
        stdpath("config") .. "/neoagent/skills",
      },
      project_dirs = { ".agents/skills" },
    },
    persistence = {
      enabled = true,
      workspace_settings = true, -- name-scoped model/thinking, shared UI
      directory = stdpath("state") .. "/neoagent/workspaces",
    },
    ui = {
      position = "auto",      -- auto/left/right/top/bottom/center
      width = nil,
      height = nil,
      margin = 1,
      input_height = 7,
      scroll_on_submit = true,
      scroll_on_transcript_leave = true,
      scroll_on_reopen = true,
      completion = {
        sources = { "files" },
      },
      border = "rounded",
      mappings = {},
    },
  }
<
CONTROLLERS AND VIEWS                                *neoagent-controllers*

`setup(opts)` constructs the built-in `Neo` and `Chat` Controllers, installs
their shared default Window, and returns `Neo`. `Neo` uses the configured
system prompt, resources, and tools. `Chat` shares the model and provider
configuration while using an empty system prompt and tool list; AGENTS.md and
skill discovery are disabled for it. Commands and top-level convenience
functions target the active Controller in that Window. `new(opts)` constructs
an independent Controller:
>lua

  local neoagent = require("neoagent")
  local coding = neoagent.setup({ name = "Coding" })
  local reviewer = neoagent.new({
    name = "Review",
    tools = require("neoagent.tools").read_only(),
    persistence = { enabled = false },
    system_prompt = "Review without editing.",
  })
  local window = neoagent.new_window({
    controllers = { coding, reviewer },
    ui = { position = "left" },
  })
  neoagent.set_default_window(window)
<
Each Controller owns its Model selection, Session, Workspace, and Run. Its
operations include `prepare`, `send`, `steer`, `dequeue_steering`, `stop`,
`new_session`, `resume`, model and thinking methods, login and logout methods,
`subscribe`,
`snapshot`, `get_session`, `get_model`, `config`, `is_running`, and `destroy`.
`send` starts an interaction while idle and queues steering while working.
Multiple Controllers can run concurrently. Controllers with a shared
persistence directory use the same cwd-hashed Workspace while retaining
independent Sessions and steering queues.

Window Controllers require unique, non-empty names. A Controller name is its
View label and workspace-settings scope. Renaming it selects a fresh scope;
matching names in separate Windows share one.

A Window owns one View and selects an attached Controller. Its operations are
`open`, `close`, `toggle`, `is_open`, `active`, `controllers`, `select`,
`cycle`, `set_input`, `input_history`, `add_input_history`,
`select_input_history`, and `destroy`. It retains input drafts per Controller
and shares workspace input history across Controllers. Selection rehydrates
compaction-aware transcript messages and transient events while every
Controller Run continues independently. Each Controller's initial Session is
created on its first send.

`set_default(controller)` points commands and top-level functions at an
existing Controller and returns the previous active Controller.
`set_default_window(window)` installs an assembled Window and returns the
previous Window. `default()` and `default_window()` return the active Controller
and command-facing Window. Another `setup()` destroys the current Window and
the Controllers owned by an earlier `setup()` call. An active Run on either
owned Controller blocks the replacement.

`controller:subscribe(callback)` publishes compaction-aware transcript
messages, context, normalized agent events, and completion updates. Its
unsubscribe function detaches the callback. `controller:snapshot()` lets a
newly attached consumer reconstruct the current transcript and partial Run.

The `view` option is a factory receiving `config`, `window`, `on_submit`,
`on_stop`, `on_dequeue_steering`, `on_input_history`, `on_select_history`,
`on_cycle_thinking`, `on_cycle_agent`, `on_select_model`, `on_resume_session`,
and `on_position_change`. It returns a passive View with `open`, `close`,
`is_open`, `destroy`, `get_input`, `set_input`, `set_messages`, `set_context`,
`apply`, and `finish`.

The final model registry composes the defaults with `providers`. Provider and
model tables recursively override matching defaults. Use `false` to remove a
default provider or model: >lua

  require("neoagent").setup({
    providers = {
      openai = {
        api_key = function() return vim.env.MY_OPENAI_API_KEY end,
        models = {
          ["gpt-5.4"] = {
            thinking = {
              minimal = false,
              high = { body = { reasoning = { effort = "high" } } },
            },
          },
          ["gpt-4"] = false,
        },
      },
      ["openai-codex"] = false,
    },
  })
<
Set `default_registry = false` to use only the user `providers` table. Built-in
providers with `auth` or `api_key` appear only when those credentials resolve.
OpenAI, Anthropic, DeepSeek, and Z.AI accept stored API keys and read
`OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `DEEPSEEK_API_KEY`, and `ZAI_API_KEY`
as ambient fallbacks. `anthropic-plan` accepts stored OAuth or
`ANTHROPIC_OAUTH_TOKEN`. Stored credentials own a provider until logout.
Providers without either are always available.
The untouched defaults are returned by
`require("neoagent.registry").defaults()`.

The built-in prompt follows the active tools and includes the current working
directory. A string replaces it completely. Append instructions by composing
the exported default in a callback: >lua

  local default_prompt = require("neoagent.system_prompt").default

  require("neoagent").setup({
    system_prompt = function(context)
      return table.concat({
        default_prompt(context),
        "Run the project formatter after editing Lua files.",
      }, "\n\n")
    end,
  })
<
The callback context contains `session`, `model`, `workspace`, the submitted
`prompt`, the active `tools`, and discovered `agents` and `skills`. A custom
value replaces the base prompt, but contextual resources are appended unless
their configuration is disabled.

The default coding preset includes the `read_agent_documentation` tool. Its
short description tells the model to call it only for Neoagent configuration,
API, UI, or extensibility questions. The result contains a concise composition
guide, Lua examples, and absolute paths to the installed documentation, source,
active init file, and Neovim configuration directory.

The `tools` option selects exactly those tools and overrides the default coding
preset. To keep the four coding tools without the documentation tool: >lua

  require("neoagent").setup({
    tools = {
      require("neoagent.tools.read_file").new(),
      require("neoagent.tools.write_file").new(),
      require("neoagent.tools.edit_file").new(),
      require("neoagent.tools.shell").new(),
    },
  })
<

Use `tools = {}` for a tool-free chat composition.

Custom compositions can add the tool directly with: >lua

  require("neoagent.tools.read_agent_documentation").new()
<

Neoagent integrations are ordinary Lua modules loaded explicitly by Neovim
configuration. They can construct independent Controllers with `new()`, attach
them to a shared Window with `new_window()`, and install that Window with
`set_default_window()`.

AGENTS.MD AND SKILLS                                  *neoagent-context-files*

The default controller loads the global file
`stdpath("config") .. "/AGENTS.md"`, followed by every `AGENTS.md` from the Git
root through the active Workspace. Without a Git root it walks from the
filesystem root. Existing files enter the prompt in broad-to-specific order.

Agent Skills are `SKILL.md` files with YAML frontmatter containing `name` and
`description`. Neoagent recursively scans these defaults:

  ~/.agents/skills
  stdpath("config") .. "/neoagent/skills"
  <each project ancestor>/.agents/skills

Later global locations override earlier skills with the same name. Project
skills override global skills, and nearer project skills override broader
ones. Only skill metadata and the `SKILL.md` path enter the prompt. The model
loads full instructions on demand with `read_file`, so skills are included only
when that tool is active. Relative references inside a skill resolve from the
directory containing its `SKILL.md`.

Replace the configured lists to customize discovery. An empty list disables
that class of locations. Set `agents = false` or `skills = false` to disable a
resource type completely. The independent `neoagent.agents` and
`neoagent.skills` modules expose `discover(opts)` and `format(resources)` for
custom workflows; discovery results include non-fatal `diagnostics`.

Provider, model, and Model:stream() options may each define `request_opts` as
a table or callback. Layers merge in that order. Only `url`, `headers`, and
`body` are accepted. Headers and body merge recursively, lists replace lists,
and header keys are case-insensitive.

Models expose optional request-time thinking controls through a `thinking`
table. Keys use the order `off`, `minimal`, `low`, `medium`, `high`, `xhigh`,
and `max`; values are request_opts tables or callbacks. Missing keys are
unsupported and `false` removes an inherited level. The default controller
passes the selected layer through `model_options.request_opts`; the Model and
agent loop do not interpret thinking levels.

WORKSPACE SETTINGS                                  *neoagent-workspace-settings*

The setup values are the base configuration. With persistence and
`workspace_settings` enabled, Controllers recursively merge name-scoped model
and thinking overrides plus the shared UI position from: >text

  workspaces/<sha256-of-canonical-root>/settings.json
<
The file uses this shape: >json

  {
    "ui_position": "right",
    "controllers": {
      "Neo": {
        "default_model": { "provider": "openai-codex", "model": "gpt-5.5" },
        "default_thinking_level": "high"
      },
      "Chat": {
        "default_model": { "provider": "openai", "model": "gpt-5.4" },
        "default_thinking_level": "medium"
      }
    }
  }
<
Names are case-sensitive. An unnamed standalone Controller uses the `default`
scope. Top-level model and thinking values act as shared fallbacks. Model,
thinking, and UI dock changes update the file atomically. A resumed session's
latest Pi model and thinking entries take precedence for that session. Every
Controller can list and resume every Session in the shared workspace session
directory. Reading settings and opening an empty Session create no files.

The bundled Window keeps its 100 most recent accepted inputs in: >text

  workspaces/<sha256-of-canonical-root>/input-history.jsonl
<
Each line is a JSON string, ordered from oldest to newest. The file preserves
multiline inputs, collapses consecutive duplicates, and is shared by every
Controller in the workspace. It is written atomically whenever persistence is
enabled. Input history remains active when `workspace_settings` is false.

The workspace settings layer is also usable independently: >lua

  local settings = require("neoagent.workspace_settings").new({
    directory = vim.fn.stdpath("state") .. "/my-plugin/workspaces",
    root = vim.fn.getcwd(),
  })
  local effective, overrides = assert(settings:merge(defaults))
  assert(settings:update({ my_option = { enabled = true } }))
  assert(settings:write(overrides)) -- replace all local overrides
<
Set `persistence.workspace_settings = false` to retain JSONL sessions while
keeping the `init.lua` model, thinking, and UI position defaults authoritative.

3. COMMANDS                                                   *neoagent-commands*

  :Neoagent                       Toggle the floating UI.
  :NeoagentCycle                  Select the next Controller.
  :NeoagentNew                    Use a new empty UI Session.
  :NeoagentResume [path]          Resume a stored Session or select one.
  :NeoagentStop                   Cancel the active UI Run.
  :NeoagentModel [provider/model] Select a configured Model or choose one.
  :NeoagentThinking [level]       Set a thinking level or cycle to the next.
  :NeoagentLogin [method]         Store an API key or authenticate an account.
  :NeoagentLogin!                 Cancel an active login.
  :NeoagentLogout [method]        Remove a stored credential or select one.
  :NeoagentCompact [instructions] Compact the active branch context.
  :NeoagentBranch [entry-id]      Move the leaf or select a branch point.
  :NeoagentFork [entry-id]        Fork before a user message or select one.

No global mappings are installed.
The argument-free Session, Model, login, branch, and fork selectors and the
input-history mapping use `vim.ui.select`, allowing UI providers such as
Telescope's `ui-select` extension to render each picker. Resume entries use the
session name or first user message, include message count and relative
activity, and group linked forks below their parent. Forking restores the
selected user message to the input buffer for editing.

4. UI                                                               *neoagent-ui*

The input and transcript are real, focusable Neovim buffers. Input opens in
Insert mode. The transcript supports normal search, motions, Visual selection,
and yank. An accepted submission scrolls the transcript to the bottom by
default. Leaving the transcript window and reopening the UI do the same. Set
`ui.scroll_on_submit`, `ui.scroll_on_transcript_leave`, or
`ui.scroll_on_reopen` to false to preserve its position for that action.
Assistant text renders Markdown headings, emphasis, links, lists,
quotes, fenced code, rules, and tables. Thinking is muted and italic. Streaming
output includes partial thinking, text, semantic tool arguments, execution
updates, results, cancellation, and errors. When the selected Model declares a
`context_window`, the footer displays used and total context tokens. It uses
the latest positive provider usage plus an estimate for trailing messages, and
estimates the projected messages when provider usage is unavailable.

User messages have a distinct padded background. Tool cards use separate
pending, successful, and failed backgrounds. Unstyled content maps NormalFloat
to Normal so it inherits the editor background. Built-in tool results use
semantic summaries. Read output shows at most ten lines until tool output is
expanded. Transcript text updates wait for pending operators and active Visual
selections, preserving register, operator, and selection state while streaming.

Enter queues steering while the active Controller is working. The queue appears
in the transcript and drains one message after the current assistant response
and tool calls. Alt-Up restores all pending steering messages to the
input. Ctrl-C restores the queue and cancels the active Run; while idle it
clears the draft.

Insert-mode Tab opens Neovim's popup menu with filename matches relative to the
current working directory and advances through them. Enter accepts the selected
match without submitting the prompt. Up, Down, Ctrl-K, and Ctrl-J navigate an
open completion menu. Set `ui.completion` to false, use an empty
`ui.completion.sources` list, or change `ui.mappings.complete` to configure the
behavior. `files` is the supported completion source.

Accepted input is shared by every Controller in the workspace. Insert-mode Up
and Down or Ctrl-K and Ctrl-J browse the most recent 100 entries. Multiline
cursor movement takes precedence until the cursor reaches the first or last
line, and returning past the newest entry restores the previous draft. Ctrl-R
opens a searchable history selector and places the selected entry in the
active draft.

Default mappings:

  <CR>        Accept popup completion, or send from input in Normal or Insert
              mode.
  <Tab>       Open filename completion or select the next popup item in Insert
              mode.
  <C-c>       Interrupt the active Run and restore queued steering; clear the
              draft while idle.
  <C-w>w or <C-w><C-w>
              Alternate between input and transcript in Normal, Insert, or
              Visual mode.
  <Esc><Esc>   Hide the UI from input Normal mode.
  <C-d>       Hide the UI when the input is empty.
  <C-o>       Expand or collapse tool output and compaction summaries.
  <S-Tab>     Cycle through supported thinking levels.
  <A-n>       Cycle Controllers (input Normal/Insert; transcript Normal).
  <A-m>       Select a model (input Normal/Insert; transcript Normal).
  <A-r>       Resume a session (input Normal/Insert; transcript Normal).
  <Up>/<Down> or <C-k>/<C-j>
              Browse workspace input history from Insert mode.
  <C-r>       Select workspace input history (input Normal/Insert; transcript
              Normal).
  <A-Up>      Restore queued steering messages to the input.
  <C-w>H/J/K/L
              Dock left, bottom, top, or right.
  <C-w>=      Center the UI.
  q           Hide the UI from the transcript.

Set a value in `ui.mappings` to an lhs string or list of strings, or to `false`
to disable it.
The centered layout uses 95% of the editor width and height by default. Dock
mappings save the selected position when workspace settings are enabled.
Card colors are customizable through `NeoagentUserBackground`,
`NeoagentToolPendingBackground`, `NeoagentToolSuccessBackground`, and
`NeoagentToolErrorBackground`. Markdown highlight groups begin with
`NeoagentMarkdown`. All Neoagent highlights use Neovim's `default` behavior.

5. CORE APIS                                                   *neoagent-core-api*

The direct Model constructor requires no setup: >lua

  local model = require("neoagent.api.openai_completions").new({
    provider = "local",
    model = "coder",
    base_url = "http://127.0.0.1:8080/v1",
    api_key = nil,                 -- string or callback
    context_window = nil,          -- enables bundled context usage display
    max_output_tokens = nil,
    request_opts = nil,
    request_opts_layers = nil,
  })

  local run = model:stream({
    messages = messages,
    system_prompt = nil,
    tools = {},
    request_opts = nil,
    on_event = function(event) end,
    on_done = function(result) end,
  })
<
The OpenAI Chat Completions adapter requests a final streamed usage chunk with
`stream_options.include_usage = true`. Provider, model, and call-level
`request_opts` may override that request field.

Use `require("neoagent.api.anthropic_messages").new` for an Anthropic Messages
endpoint with the same Model interface. It sends `POST /messages`,
`anthropic-version: 2023-06-01`, `x-api-key` authentication for API keys,
Bearer authentication for `sk-ant-oat` OAuth tokens, Anthropic content and
tool blocks, and `max_tokens`. The default
`max_output_tokens` is 4096. Signed and redacted thinking blocks are replayed;
unsigned thinking is replayed as text. Streamed message, content, usage, stop,
and error events are normalized into Neoagent messages and events.

Use `require("neoagent.api.openai_responses").new` for the same Model interface
with a Responses endpoint. Its additional constructor options are
`reasoning`, `reasoning_effort`, `reasoning_summary`, and
`reasoning_context`. Set `reasoning_summary = "none"` to omit the summary
parameter. `require("neoagent.api.openai_codex_responses").new` also accepts
`responses_lite = true`, which sends tools and instructions as leading
developer input items and sets the Codex Lite header. It exposes the Codex SSE
request profile without performing login or credential lookup, accepts
`request_max_retries` (default 4), and accepts an `on_diagnostic(event)`
callback. Direct construction writes no diagnostic file.

Authentication is separately composable: >lua

  local manager = require("neoagent.auth").new({
    methods = { my_plan = my_login_method },
    store = my_credential_store,
  })
  local authenticated_model = manager:wrap(model, "my_plan")
<
Credentials have one of these tagged forms: >lua

  { type = "api_key", key = "...", env = { OPTIONAL = "value" } }
  { type = "oauth", access = "...", refresh = "...", expires = 0 }
<
Untagged OAuth values are accepted as OAuth credentials. A method has an
optional `type`, `name`, `login(interaction)`, and `request_opts(credential)`.
OAuth methods also have `refresh(credential)`. Login and refresh return Runs.
The interaction prompt types are `text`, `secret`, `select`, and `manual_code`;
the bundled Controller uses a masked command-line prompt for `secret`.

`require("neoagent.auth.api_key").new({ name = "Example API key" })` creates a
Bearer-key method. Its optional `request_opts(credential)` callback can derive
another header profile.

A store implements `read(id)` and `write(id, credential)`. `modify(id, fn)`
serializes login and refresh when present. `list()` returns only credential IDs
and types, and `delete(id)` removes a credential under the same lock. The
bundled store implements all five operations atomically with private file and
directory modes. `manager:list_credentials()` exposes secret-free metadata and
`manager:logout(id)` returns a cancellable Run.

The wrapper resolves credentials when `Model:stream()` starts and has no UI,
Session, or tool dependency. `manager:wrap(model, id, { optional = true })`
runs the underlying Model when no stored credential exists, allowing its
ambient API-key callback to supply authentication. A stored credential remains
authoritative and malformed stored values produce an auth error.

Model events are `text_delta`, `thinking_delta`, `tool_call_delta`, and
`usage`. A Model may emit `{ type = "provider_status", text = "..." }`; the
Controller retains it as transient provider status. A successful result has
`{ ok = true, message, text }`; a failure has
`{ ok = false, error, message? }`. Non-empty partial output is retained in the
optional failure message.

A Run supports: >lua

  run:cancel()
  run:is_done()
  run:is_cancelled()
  run:result()
  run:await()       -- only inside neoagent.async.run()
<
Callbacks run on Neovim's main loop and completion occurs exactly once.

The tool loop is: >lua

  require("neoagent.agent").run({
    model = model,
    messages = messages,
    system_prompt = nil,
    tools = {},
    execute_tool = nil,
    context = nil,
    model_options = nil,
    on_event = nil,
    on_done = nil,
  })
<
It copies input messages, performs no global model lookup, and uses exactly the
tools passed by the caller. The loop continues while the model requests tools
or steering supplies another turn. A final assistant response or cancellation
ends the Run.

To bridge a callback API inside a tool or executor: >lua

  local value = require("neoagent.async").await(function(done)
    start_operation(function(err, result)
      if err then done.reject(err) else done.resolve(result) end
    end)
    return function() cancel_operation() end
  end)
<

6. TOOLS AND SESSIONS                                    *neoagent-composition*

A tool is `{ name, description, input_schema, execute }`. `execute(arguments,
ctx)` returns `{ content = { ... }, isError?, details? }`; content blocks are
text or base64 images. Replace or decorate `execute_tool(tool, arguments, ctx)`
to add approval, logging, a native sandbox, or subagent behavior.

Bundled presets: >lua

  require("neoagent.tools").coding()    -- read/write/edit/shell/Neoagent docs
  require("neoagent.tools").read_only() -- read/grep/find
  require("neoagent.tools").all()
<
File tools operate on disk, never loaded buffers.

`require("neoagent.session").new()` is an in-memory, tool-free message owner.
It may receive `{ store = store }` or `{ messages = messages }`, but not both.
The default JSONL Store is created with: >lua

  local store = require("neoagent.storage").new({
    directory = directory,
    cwd = cwd,
  })
<
`directory` is the workspaces root. The store writes below
`<directory>/<sha256-of-canonical-cwd>/sessions/`. It uses the full Pi v3
append-only JSONL tree format and creates no file until the first accepted
message. It supports message, model, thinking, active-tool, compaction,
branch-summary, custom, custom-message, label, session-info, and leaf entries.
All Controllers share this session directory and may resume any stored path.
Use `storage.open(path)` to resume.

Tree-aware Session methods are: >lua

  session:messages()                 -- active branch for consumers
  session:context_messages()         -- compacted LLM projection
  session:entries()
  session:entry(id)
  session:leaf_id()
  session:path([id])
  session:state()
  session:move_to(id[, summary])
  session:append_entry(type, values)
  session:label(id)
  session:name()
<
`move_to()` changes the active leaf; the next append creates a branch child.
The optional summary creates a Pi `branch_summary` entry. Fork a linked child
session with: >lua

  local child = require("neoagent.storage").fork(store_or_path, {
    directory = directory,
    entry_id = user_entry_id,
    position = "before", -- or "at"
  })
<
Omitting `entry_id` copies the complete append-only file. The default
Controller exposes `branch`, `select_branch`, `fork`, and `select_fork`.

`store:info()` returns the path, ID, cwd, optional name and parent, creation
timestamp, latest message activity, message count, and first user message.
`storage.list_sessions(directory, cwd)` returns these values in recent-activity
order. The resume selector groups linked files through `parentSession`.
Branches within a file remain available through `:NeoagentBranch`.

CONTEXT COMPACTION                                  *neoagent-compaction*

Automatic compaction starts after an interaction crosses the Model's context
threshold, before a prompt when resumed context already exceeds it, and once
for a provider context-overflow error. It asks the active Model for a
structured summary with an empty tool list and appends a Pi `compaction`
entry. Subsequent requests and Controller transcript snapshots contain that
checkpoint and the retained recent entries. The built-in View renders an
expandable compaction card followed by retained and subsequent messages;
`<C-o>` toggles the complete summary. Prefix entries remain available through
`session:messages()`, `session:path()`, the persisted JSONL tree, and branch
navigation. Repeated compactions update the previous checkpoint; oversized
turns receive a separate prefix summary to preserve tool-call boundaries.

The threshold is `context_window - reserve_tokens`. Recent retention targets
`keep_recent_tokens`. Defaults are reduced proportionally for smaller Model
windows. Set `compaction = false` to disable all Controller compaction or
`compaction.auto = false` to retain manual and overflow compaction. A custom
`compaction.run(opts)` returns a cancellable Run with `summary`,
`first_kept_entry_id`, `tokens_before`, and optional `usage` and `details`.
The `neoagent.compaction` module exposes `settings`, `estimate_context`,
`should_compact`, `prepare`, `serialize`, and `run` for direct composition.

Session adapters: >lua

  require("neoagent.chat").send(session, prompt, opts)
  require("neoagent.chat").run(session, prompt, opts)
  require("neoagent.chat").continue(session, opts)
<
`continue()` uses the projected context without appending another user
message. `run()` and `continue()` forward an optional `get_steering_messages`
callback to the agent loop. It returns user-message tables to inject after the
current assistant response and tool calls. Accepted messages are persisted.
Session adapters allow one active mutation per Session.
The top-level `require("neoagent")` module constructs Controllers with `new`,
constructs shared Windows with `new_window`, owns the replaceable default
Window, and forwards commands and convenience operations to its active
Controller.
